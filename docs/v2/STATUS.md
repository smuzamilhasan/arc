## 4. Voice extraction тАФ ЁЯЯв

5-pass pipeline: deterministic stats (sentence/lexicon/punctuation) + 4 LLM passes (voice features, story candidates, reference candidates, worldview hypotheses). Outputs a `ProfilePatch` with `evidence_sample_ids`. Refuses below 10 samples or on multi-author variance.

## 5. Onboarding (conversational) тАФ ЁЯЯв

Per-turn stateless `RoleContract`; `OnboardingSessionService` holds state in `onboarding_sessions`. Output is a discriminated turn (`question | patch | wrap`). Playbook drives slot progression. Confirms extractor candidates before committing.

## Known gotchas (do not re-learn the hard way)

1. **`/api` double-prefix.** Routes mount via `app.use("/api", router)`; v2 route paths must omit `/api` (use `/v2/...`). A leading `/api/v2` becomes `/api/api/v2` тЖТ 404.
2. **OpenAI strict mode compat** (`agents-v2/llm/openaiAdapter.ts`): strip unsupported `format` (e.g. `uri`); all properties required + optionalтЖТnullable; wrap non-object roots (discriminated unions) in `{response: тАж}` and unwrap; strip null before Zod re-validate. **`z.record()` emits `propertyNames`, which strict mode rejects тАФ use a JSON string field and `JSON.parse`.**
3. **Adding profile fields = no DB migration** if the key lives inside an existing JSONB layer (Zod-only change). A new column *does* need a migration.
4. **Re-running calibration won't re-charge Apify** unless `force:true` тАФ it reuses cached `voice_samples`.
5. **Railway** auto-deploys on push to `main` (project `mindful-ambition`, service `practical-purpose`). Wait ~3.5 min before verifying.
6. **Devanagari vowel signs are Unicode marks** (`\p{M}`), not letters тАФ any tokenizer over Urdu auto-captions must include them or it truncates words.
